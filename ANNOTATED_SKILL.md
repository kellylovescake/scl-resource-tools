# Annotated Bibliography Skill

Produces a branded .docx annotated bibliography for Sunflower County Library System.
This is a **separate mode** from the standard Bibliography Builder — use when patrons
need guidance, not just a title list.

## What this produces

Same layout and branding as the standard Bibliography Builder, plus:
- A one-sentence annotation beneath each entry's author/collection line
- Annotation style: 9pt italic, muted gray (`#888888`), indented 0.2"
- Annotations are generated by Claude from title + author, then reviewed by Kelly
  before the document is built

## Generation script

```bash
node scripts/generate_annotated_list.js '<json>'
```

JSON shape is identical to `generate_list.js` with one addition — each book
object may include an `annotation` field:

```json
{
  "title": "Understanding Diabetes",
  "ageGroup": "for Adults",
  "preparedBy": "Kelly Waddell, MLIS",
  "dateStr": "03/18/2026",
  "books": [
    {
      "title": "The Diabetes Code",
      "author": "Fung, Jason",
      "collection": "Adult Non-Fiction",
      "callNumber": "616.4 F",
      "annotation": "A physician explains how dietary changes can reverse type 2 diabetes without medication."
    }
  ],
  "digitalResources": ["Hoopla – ebooks, audiobooks, and comics"],
  "outputPath": "/mnt/user-data/outputs/Understanding_Diabetes_for_Adults_20260318.docx"
}
```

`annotation` is optional per entry — entries without it render without the annotation line.

## Conversational workflow

### Step 1 — Receive the book list

Same as Bibliography Builder. Accepts ILS export format or plain text.

---

### Step 2 — Collaborate on a title

Same as Bibliography Builder. Suggest 2–3 patron-friendly options, wait for Kelly to choose.

---

### Step 3 — Age group

Same as Bibliography Builder.

---

### Step 4 — Prepared by

Same as Bibliography Builder. Always ask; never default.

---

### Step 5 — Generate and present annotations for review

After collecting the book list, generate one-sentence annotations for each entry
using Claude's knowledge of the title and author.

**Annotation guidelines:**
- Exactly 1 sentence
- Plain declarative — no "This book…" opener, no marketing language
- Focus on what the patron will find inside: topic, angle, audience, usefulness
- Under 20 words is ideal; 25 is the hard cap
- Do not editorialize ("excellent," "must-read")

Present annotations in a numbered list matching the book order:

> Here are the annotations I've drafted. Edit any that need adjusting — just
> reply with the number and your revised text:
>
> 1. **The Diabetes Code** — A physician explains how dietary changes can reverse
>    type 2 diabetes without medication.
> 2. **Think Like a Pancreas** — A practical guide to blood sugar management
>    written by a diabetes educator who lives with the condition.

Wait for Kelly to approve or correct before proceeding to Step 6.
If she says "looks good" or similar, proceed with all annotations as-is.

---

### Step 6 — Digital resources

Same as Bibliography Builder. Always prompt; never skip. SCL offers Hoopla only.

---

### Step 7 — Generate the document

Call the generation script with the approved annotations included in each book object.
Output path format: `{Title}_{AgeGroup}_{YYYYMMDD}.docx`

---

### Step 8 — Deliver

Use `present_files` to share the .docx.
Mention they can print from Word or export to PDF via File → Save As.

## Design spec changes from standard Bibliography Builder

| Element | Value |
|---|---|
| Annotation text | 8.5pt (size: 17) italic, `#888888` |
| Annotation indent | 0.2" left |
| Entry page breaking | `cantSplit: true` on every `TableRow` — entries never split across pages; Word inserts a page break before an entry that won't fit |

All other design values are inherited from the standard Bibliography Builder spec.
